Soluzione definitiva per il problema di cache / file pubblicato

Questa pagina riassume il fix corretto per GitHub Pages quando il sito continua a mostrare il vecchio app.js.

1) Verifica il file caricato

Nel tuo index.html il file caricato è js/app.js, quindi devi aggiornare proprio quello.

<script src="js/app.js"></script>

2) Sintomo tipico

  • La pagina mostra ancora il vecchio popup o vecchi pulsanti.
  • Il browser sembra ignorare il nuovo codice anche dopo il refresh.
  • GitHub Pages potrebbe servire una build vecchia o un branch errato.

3) Fix definitivo consigliato

  1. Apri il repository su GitHub e sostituisci js/app.js.
  2. Fai un commit con un messaggio chiaro, ad esempio fix: remove broken report modal.
  3. Fai push sul branch pubblicato da GitHub Pages.
  4. In Settings → Pages, disattiva e riattiva GitHub Pages se la build resta bloccata.
  5. Fai git commit --allow-empty -m "Trigger rebuild" e pushalo per forzare una nuova build.
  6. Apri il sito in incognito oppure con DevTools aperti e Disable cache.

4) Se vuoi evitare il problema in futuro

  • Rinomina il file con una versione, per esempio js/app.v2.js, e aggiorna l'index.html.
  • Non usare più copie dello stesso file in percorsi diversi.
  • Quando cambi una grossa porzione di logica, cambia anche il nome del file o aggiungi un query string tipo js/app.js?v=2.
